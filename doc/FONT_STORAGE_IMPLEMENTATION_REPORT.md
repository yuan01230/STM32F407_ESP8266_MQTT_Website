# 字库加载与混排显示改造报告

## 1. 背景与目标

当前工程已完成 SD 读写与信息显示测试，但中文显示受限：原有 `LCD_ShowString` 仅支持 ASCII。

本次改造目标：

1. 采用 `GB2312 + HZK16` 方案支持中文显示。
2. 支持字库从 SD 卡导入到外挂 Flash（EN25Q128）。
3. 运行期从外挂 Flash 读取字模，降低对 SD 在线依赖。
4. 提供中英文混排显示接口，兼容现有 ASCII 显示逻辑。

---

## 2. 方案决策

采用方案：**SD 作为字库来源 + 外挂 Flash 作为运行字库 + GB2312 统一编码 + 混排封装接口**。

原因：

1. SD 更新字库方便，适合维护升级。
2. 外挂 Flash 随机读取稳定，运行期性能和可靠性更好。
3. GB2312 与 HZK16 直接匹配，MCU 端实现复杂度低。
4. 混排接口可在同一行同时显示 ASCII 与中文，便于菜单和状态页统一展示。

## 4. 实施过程记录

### 步骤 1：新增字库存储模块（font_storage）

实现内容：

1. 定义外部 Flash 字库头结构（magic/version/size/crc）。
2. 启动时读取并校验字库头，判断字库是否可用。
3. 提供从 SD 文件 `0:/HZK16` 导入到外挂 Flash 的流程：
   - 分块读取 SD 文件；
   - 擦除目标扇区；
   - 逐块写入 EN25Q128；
   - 计算并保存 CRC 与头信息。
4. 提供按偏移读取 16x16 字模（32 字节）接口。

核心接口：

1. `FontStorage_Init()`
2. `FontStorage_IsReady()`
3. `FontStorage_ImportFromSD(const char *path)`
4. `FontStorage_ReadGlyph16(uint32_t glyph_offset, uint8_t *glyph_buf32)`

---

### 步骤 2：新增编码解析模块（font_codec）

实现内容：

1. 解析输入文本中的 ASCII 与 GB2312 双字节字符。
2. 对中文字符计算 HZK16 偏移。

偏移公式：

`((qh - 0xA1) * 94 + (wh - 0xA1)) * 32`

核心接口：

1. `FontCodec_ParseGB2312(const uint8_t *text, FontCodecToken *token)`

---

### 步骤 3：在 TFTLCD 中增加混排显示接口

修改点：

1. 在 `tftlcd.h` 增加对外接口：
   - `LCD_ShowTextMixed(uint16_t x, uint16_t y, uint16_t width, uint16_t height, const uint8_t *text);`
2. 在 `tftlcd.c` 中实现：
   - ASCII：复用现有 `LCD_ShowChar` 路径；
   - 中文：`GB2312 解析 -> Flash 读 32 字节字模 -> 16x16 点阵绘制`；
   - 字模读取失败：绘制红色占位框；
   - 处理换行、区域宽度换行、显示边界。

---

### 步骤 4：主流程接入（main.c）

修改点：

1. 引入 `EN25Q128` 与 `font_storage` 头文件。
2. 启动阶段增加 `EN25QXX_Init()`。
3. 新增 `Font_System_Init(sd_ready)`：
   - 若 Flash 字库可用：直接使用；
   - 若不可用且 SD 已挂载：尝试导入 `0:/HZK16`；
   - 串口打印导入成功/失败日志。
4. 在主菜单增加一行混排演示文本：
   - `GB2312 Demo: 中文`（中文使用 GB2312 字节序列）。

---

### 步骤 5：构建与文档更新

1. `CMakeLists.txt` 增加 `font_storage` 与 `font_codec` 源文件。
2. `README.md` 追加字库存储方案和使用说明。

## 5. 当前行为说明

1. 系统启动后先检查外挂 Flash 是否已有有效字库。
2. 若无有效字库且 SD 可用，则自动导入 `0:/HZK16`。
3. 运行期间中文显示从外挂 Flash 读取，不依赖 SD 在线。
4. 混排接口可在一行内显示中英混合文本。

---

## 6. 使用前提与注意事项

1. SD 根目录需存在字库文件：`HZK16`。
2. 推荐 `HZK16` 标准大小：`282752` 字节（94x94x32）。
3. 输入中文字符串需为 **GB2312 编码**。
4. 首版按 16x16 字模实现，后续可扩展 24x24/32x32。

## 10. 上板前后操作步骤（字库准备与验证）

### 10.1 准备字库文件

1. 准备 `HZK16` 字库文件（GB2312）。
2. 文件名使用：`HZK16`（无扩展名）。
3. 推荐标准大小：`282752` 字节（94x94x32）。

### 10.2 拷贝到 SD 卡

1. 将字库文件放入 SD 卡根目录。
2. 目标路径应为：`0:/HZK16`。
3. 确认 SD 文件系统可正常挂载（FAT/FAT32）。

### 10.3 上电启动验证

1. 观察串口日志是否出现以下过程：
   - 外挂 Flash 字库检查；
   - 若未就绪则从 SD 导入；
   - 导入成功或失败码打印。
2. 若已导入成功，后续重启应可直接从 Flash 使用字库。

### 10.4 LCD 页面验证

1. 检查“中文行”显示是否正确。
2. 检查“英文行”显示是否正确。
3. 检查“混排行”显示是否正确。
4. 检查状态行：`Font: Ready in Flash`。

### 10.5 按键功能验证

1. `KEY0`：切换文字颜色模式。
2. `KEY1`：重绘测试页面。
3. `KEY2`：清屏后重绘测试页面。
4. `KEY_UP`：强制从 SD 重新导入 `HZK16` 并刷新状态。

### 10.6 异常排查建议

1. 若中文显示为占位框：
   - 检查 `HZK16` 是否存在于 `0:/`；
   - 检查文件大小与编码是否正确；
   - 检查导入日志是否报错。
2. 若 `Font: Not Ready`：
   - 先确认 SD 是否挂载成功；
   - 使用 `KEY_UP` 手动重导入；
   - 重启后再次检查状态。
