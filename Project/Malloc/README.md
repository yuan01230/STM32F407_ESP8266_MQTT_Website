# STM32 内存管理模块 (Malloc) - 演示项目说明

## 1. 项目简介
本项目基于 **STM32F407** 平台，实现了一套轻量级、高性能的**多内存池动态内存管理方案**。它不仅能统一管理内部 SRAM、外部 FSMC SRAM 以及 CCM RAM，还提供了一个直观的 **LCD 交互界面**，用于实时演示内存的申请、释放及使用率监控。

## 2. 核心特性
*   **多物理内存池支持**：
    *   **SRAMIN**: 内部 SRAM (100KB)，速度最快。
    *   **SRAMEX**: 外部 IS62WV51216 SRAM (960KB)，容量大，通过 FSMC 访问。
    *   **SRAMCCM**: CCM RAM (60KB)，专供 CPU 高速运算使用。
*   **块式分配算法**：采用 32 字节固定块大小，通过状态表（Bitmap）追踪占用情况，有效减少碎片。
*   **高性能优化**：
    *   `my_mem_set` / `my_mem_cpy` 采用 32 位字操作及循环展开技术。
    *   **懒加载初始化**：仅清零状态表，大幅缩短大容量外部 SRAM 的启动时间。
*   **交互式演示系统**：
    *   支持 **30 个并发内存槽位**，可多次申请不同地址的内存块。
    *   LCD 实时显示温湿度传感器数据及各内存池的使用率百分比。
    *   按键即时反馈申请/释放的具体内存地址。

## 3. 硬件资源与布局
| 组件 | 型号/规格 | 备注 |
| :--- | :--- | :--- |
| **主控芯片** | STM32F407ZGT6 | Cortex-M4, 168MHz |
| **外部 SRAM** | IS62WV51216 | 1MB, FSMC Bank1 Region3 (0x68000000) |
| **显示屏** | TFT-LCD | 3.5寸/4.3寸, FSMC 接口 |
| **传感器** | DHT11 | 温湿度监测 |
| **按键** | KEY0, KEY1, KEY2 | 用户交互输入 |

## 4. 演示功能操作指南

### 4.1 界面说明
程序启动后，LCD 将显示以下分区：
*   **顶部状态栏**：蓝色高亮显示当前选中的内存池（如 `Current Pool: SRAMIN`）。
*   **核心监控区**：显示当前池的总使用率（`Pool Usage`）以及最近一次操作的地址详情。
*   **环境数据区**：每秒刷新一次 DHT11 采集的温度和湿度。
*   **操作提示区**：底部红色字体标注三个按键的功能。

### 4.2 按键功能
1.  **KEY0 (申请内存)**：
    *   从当前选中的内存池中申请 **1KB** 空间。
    *   LCD 会显示 `Alloc[0]: 0x2000xxxx`，其中 `[0]` 为槽位编号。
    *   连续按压可填满 30 个槽位，观察 `Pool Usage` 逐步上升。
2.  **KEY1 (释放内存)**：
    *   释放**最近一次**申请的内存块。
    *   LCD 会显示 `Free[0]: 0x2000xxxx`，确认释放的地址。
    *   释放后该槽位变为空闲，可供再次申请。
3.  **KEY2 (切换内存池)**：
    *   在 `SRAMIN` -> `SRAMEX` -> `SRAMCCM` 之间循环切换。
    *   切换后，KEY0 将从新的物理内存区域分配空间。

## 5. 软件架构与配置

### 5.1 链接脚本配置 (.ld)
为了将巨大的内存池数组放置到指定的物理地址，必须在链接脚本中定义段：
```ld
MEMORY {
    SRAMEX (xrw) : ORIGIN = 0x68000000, LENGTH = 1024K
}
SECTIONS {
    .sramex_pool (NOLOAD) : { *(.sramex_pool) } >SRAMEX
    /* ... 其他池定义 ... */
}
```

### 5.2 CMake 构建配置
在 `CMakeLists.txt` 中确保包含 Malloc 源文件及头文件路径：
```cmake
target_sources(1_LED PRIVATE Malloc/malloc.c)
target_include_directories(1_LED PRIVATE ${CMAKE_CURRENT_SOURCE_DIR}/Malloc)
```

### 5.3 关键驱动依赖
*   **FSMC 驱动**：必须调用 `SRAM_Init()` 完成外部存储器的 GPIO 和时序配置，否则访问 `0x68000000` 会导致 HardFault。
*   **DHT11 驱动**：用于提供实时的环境监测数据，丰富演示内容。

## 6. 注意事项
1.  **CCM RAM 限制**：CCM RAM 只能通过 CPU 访问，**严禁**将其作为 DMA 传输的缓冲区。
2.  **线程安全**：当前版本未加入互斥锁。若在 RTOS 环境下使用，请在 `mymalloc` 和 `myfree` 外层添加信号量保护。
3.  **地址对齐**：所有申请的内存地址均会自动向上对齐到 32 字节边界，以确保 FSMC 访问效率。
