# 阿里云物联网仪表盘

这是一个本地运行的阿里云 IoT 可视化仪表盘示例项目，包含：

- 一个静态前端仪表盘
- 一个 Node.js / Express 后端
- 本地仿真模式
- 阿里云 OpenAPI 接入模式

项目当前既可以在没有真实设备的情况下用本地模拟数据演示，也可以在配置阿里云凭证后读取真实设备状态，并向设备发送属性控制指令。

## 项目结构

```text
aliyun-iot-dashboard/
  client/
    index.html

    style.css
  server/

  README.md
```

## 功能概览

- 实时显示 Sensor 设备的温度、湿度、光敏 ADC、电压输出
- 显示通用电能设备的电压、电流、功率和告警状态
- 支持设备切换
- 支持历史趋势图表
- 支持日志控制台
- 支持本地添加和删除设备
- 支持在前端配置阿里云 OpenAPI 凭证
- 支持向 Sensor 设备发送 LED 和电压控制命令
- 支持本地仿真模式

## 本地启动

进入服务端目录并启动：

```powershell
cd F:\antigravity_workspace\aliyun-iot-dashboard\server
npm start
```

启动后访问：

```text
http://localhost:5000
```

## 本地仿真模式

如果没有配置阿里云凭证，系统会自动进入本地仿真模式。

在这个模式下：

- 页面会持续显示模拟数据
- Sensor 设备会保持在线展示
- 控制操作只会在本地界面和服务端内存中生效
- 不会真正发送到阿里云

## 接入阿里云 OpenAPI

你可以在页面右上角打开“阿里云 OpenAPI”配置面板，填写：

- `AccessKey ID`
- `AccessKey Secret`
- `Region ID`

配置成功后，系统会尝试读取真实设备状态。

## 安全说明

当前版本是本地演示项目，不是生产级安全方案。

注意事项：

- 凭证只保存在本地运行的服务进程中
- 服务重启后，前端添加的设备和当前会话配置不会持久化
- 不要把真实生产凭证暴露在不受信任的环境中
