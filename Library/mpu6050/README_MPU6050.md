# MPU6050 DMP 移植与开发指南 (STM32F407)

本教程记录了将 MPU6050 官方 DMP (Digital Motion Processor) 驱动移植到基于 STM32F407 开发板（使用软件 I2C 通讯）的全过程。

## 一、 移植步骤回顾

### 1. 驱动库引入与组织
- **来源**: 克隆官方 eMPL (embedded Motion Processing Library) 驱动。
- **结构**: 
  - `MPU6050.c/h`: 硬件适配层（I2C 读写接口、基础初始化）。
  - `dmp/inv_mpu.c/h`: InvenSense 核心寄存器驱动。
  - `dmp/inv_mpu_dmp_motion_driver.c/h`: DMP 固件加载与数据解析。

### 2. 堆栈空间调整 (关键)
- **现象**: DMP 初始化时会申请较大的局部变量数组。如果栈空间不足（默认 1KB），会导致程序进入 `HardFault` 或初始化失败。
- **操作**: 修改 `STM32F407XX_FLASH.ld`，将 `_Min_Stack_Size` 从 `0x400` 修改为 **`0x1000` (4KB)**。

### 3. 底层接口适配 (软件 I2C)
- **I2C 读写**: 在 `MPU6050.c` 中重写 `IIC_Write_Len` 和 `IIC_Read_Len`，对接项目现有的 `SoftwareI2C` 库。
- **平台宏定义**: 在 `inv_mpu.c` 顶部定义 `i2c_write`、`i2c_read`、`delay_ms` 和 `get_ms` 等宏。
- **时间戳**: 实现 `get_tick_count` 函数，返回 `HAL_GetTick()`，供 DMP 驱动计算采样间隔。

### 4. 串口重定向 (重要稳定性修复)
- **必要性**: 使用 `printf` 时必须在 `main.c` 或相关文件中实现 `fputc` 重定向。
- **风险**: 若未重定向，程序在调用 `printf` 时可能会因为找不到底层输出函数而陷入死循环或由于标准库行为导致外设初始化顺序异常（如 LED 无法点亮）。

### 5. LCD 显示优化 (防闪烁)
- **分频刷新策略**: 
  - **传感器采样/串口输出**: 保持 **50ms (20Hz)**，确保 3D 上位机动画流畅。
  - **LCD 屏幕刷新**: 降低至 **150ms (约 7Hz)**。人眼对数字跳动的感知存在滞后，低频刷新可有效消除 LCD 局部重绘带来的闪烁感。
- **覆盖逻辑**: 在 `sprintf` 格式化字符串末尾添加空格（如 `"Pitch: %.2f    "`），利用 `LCD_ShowString` 自身的背景填充特性覆盖旧数据，避免频繁调用 `LCD_Clear`。

## 二、 注意事项

1. **静止初始化**: `mpu_dmp_init()` 执行时，必须保持开发板水平静止。DMP 会在此时进行陀螺仪自检和零偏校准。
2. **I2C 速度**: 强烈建议使用 400kHz 快速模式。DMP 的 FIFO 每次输出包含四元数、角速度等大量数据，低速 I2C 会显著阻塞主循环。
3. **坐标系对齐**: 
   - 针对 3D 仿真软件方向与实物镜像的问题，在角度换算处为 `Pitch` 和 `Roll` 添加负号。
   - `*pitch = -asin(-2 * q1 * q3 + 2 * q0 * q2) * 57.3f;`
4. **串口独占**: 使用配套 3D 软件前，务必关闭 IDE 内置串口终端。

## 三、 MPU6050 基础知识

### 1. 加速度计 vs 陀螺仪
- **加速度计 (Accelerometer)**: 测量地球重力方向。优点是长久稳定，缺点是运动瞬间噪声极大（容易受震动干扰）。
- **陀螺仪 (Gyroscope)**: 测量旋转角速度。优点是响应极快，缺点是随时间推移会产生积分漂移（静止时角度也会慢慢变）。

### 2. DMP 硬件解算原理
直接读取原始数据（Raw Data）无法直接使用。DMP 是 MPU6050 内部的一个微处理器：
- **功能**: 它能运行 InvenSense 官方的姿态融合算法。
- **产出**: 它是将加速度和陀螺仪数据在内部进行“卡尔曼滤波”级融合后，直接输出**四元数 (Quaternion)**。
- **优势**: 姿态解算不占用 STM32 的 CPU 周期，且输出的角度非常稳定，几乎没有温漂和震动噪声。

## 四、 核心接口函数

- `u8 MPU_Init(void)`: 检测 MPU6050 ID 并初始化基础寄存器。
- `u8 mpu_dmp_init(void)`: 加载 DMP 固件，这是最耗时也最容易因栈空间不足而失败的步奏。
- `u8 mpu_dmp_get_data(float *p, float *r, float *y)`: 从传感器的 FIFO 队列中取出最新的数据包并解析为欧拉角。
