# Picture

## 1. 模块定位

`Library/Picture` 目录负责当前工程中的图片预览能力。

当前项目目标不是做完整桌面级图片系统，而是在 `STM32F407 + FATFS + LCD` 场景下，稳定实现：

- 从 `SD` 卡读取图片文件
- 在文件浏览器中预览图片
- 支持常用的 `BMP` 和 `JPG/JPEG`
- 对大图进行等比缩小
- 在预览区域中居中显示

---

## 2. 当前已实现功能

### 2.1 通用文件浏览器

当前 UI 已支持从 `0:/` 根目录开始浏览：

- 同时显示目录和普通文件
- 目录中自动生成 `[..]` 返回上一级
- `KEY2` 统一负责进入目录或打开文件
- `KEY0` 在列表页返回主页面
- `KEY0` / `KEY2` 在查看页返回列表

### 2.2 运行时图片预览

当前支持的运行时图片格式：

- `BMP`
- `JPG`
- `JPEG`

图片文件在查看页会自动进入预览分支，而不是按文本或十六进制内容显示。

### 2.3 BMP 预览能力

当前 `BMP` 路径支持：

- 标准 `24-bit BMP`
- 简单兼容的 `16-bit BMP`
- 逐行读取，不整图加载
- `BGR` 到 `RGB565` 转换
- 大图等比缩小
- 小图不放大
- 居中显示到预览区域

### 2.4 JPG/JPEG 预览能力

当前 `JPG/JPEG` 路径基于 `TJpgDec` 软件解码器。

支持特点：

- 从 `SD` 卡通过 `FATFS` 读取 `JPG/JPEG`
- 运行时解码并输出到 `LCD`
- 使用 `TJpgDec` 的缩放级别：
  - `1/1`
  - `1/2`
  - `1/4`
  - `1/8`
- 对大图尽量按比例缩小
- 小图不放大
- 最终在预览区域中居中显示

---

## 3. 当前图片预览行为

当前查看页的图片预览区域是固定的，位于文件信息下方。

预览行为规则如下：

1. 如果图片比预览区域大：
- 按原始比例缩小
- 尽量完整放入预览区域
- 缩小后居中显示

2. 如果图片比预览区域小：
- 不放大
- 保持原始尺寸
- 直接居中显示

3. 每次显示新图片前：
- 先清空预览区域背景
- 避免上一张图残留像素

---

## 4. JPG 素材规范

当前工程虽然已经支持 `JPG/JPEG` 运行时预览，但不是所有 JPEG 文件都保证能正常打开。

### 4.1 推荐使用的 JPG 规格

建议图片资源统一满足以下条件：

- `Baseline JPEG`
- `RGB` / 常规照片导出模式
- 普通导出参数
- 尽量避免过于复杂的压缩选项

推荐测试规格：

- `320 x 240`
- `240 x 320`
- `240 x 240`
- 文件大小先控制在几十 KB 到几百 KB

### 4.2 不推荐直接使用的 JPG

以下 JPEG 在当前版本中可能打不开：

- `Progressive JPEG`
- `CMYK JPEG`
- 某些特殊采样方式的 JPEG
- 某些手机或网站导出的复杂编码 JPEG

这些文件即使扩展名是 `.jpg`，也可能因为编码方式不受当前解码器支持而失败。

## 5. 图片预览状态提示

当前图片预览结果会显示在：

- `Image Preview: ...`

可能出现的状态包括：

- `Image Preview: OK`
  - 图片已正常打开并显示

- `Image Preview: JPG UNSUP`
  - 该 JPG 的编码方式超出当前 `TJpgDec` 支持范围
  - 最常见原因是 `Progressive JPEG` 或特殊 JPEG 编码参数

- `Image Preview: JPG NOMEM`
  - 当前 JPEG 解码工作区内存不足

- `Image Preview: JPG FORMAT`
  - JPEG 文件结构损坏或头信息非法

- `Image Preview: OPEN FAIL`
  - 文件打开失败，常见原因是路径错误、文件不存在、SD 卡未正常挂载

- `Image Preview: READ ERR`
  - 文件读取失败

- `Image Preview: RANGE ERR`
  - 显示区域参数无效或超出 LCD 范围

---

## 6. 当前已知限制

当前版本仍有这些限制：

- 不支持 `PNG` 运行时解码
- 不支持 `GIF` 运行时解码
- 不支持小图放大
- 不支持用户交互缩放
- 不支持拖动查看大图
- `JPEG` 缩放精度不如 `BMP` 软件缩放细，因为当前依赖 `TJpgDec` 的离散缩放级别

---

## 7. 当前工程建议的图片资源策略

如果目标是让当前工程长期稳定使用，推荐：

1. `BMP`
- 用于调试、验证、教学和格式研究
- 推荐 `24-bit` 无压缩 `BMP`

2. `JPG/JPEG`
- 用于普通照片类素材
- 统一导出为 `Baseline JPEG`

3. 大批量项目资源
- 若后续追求更高稳定性和加载效率，可考虑继续转为：
  - `RGB565`
  - 或其他更轻量的 MCU 友好格式

---

## 8. 当前版本总结

截至目前，这个项目已经具备：

- 通用 `SD` 卡文件浏览器
- `.bmp` 运行时预览
- `.jpg/.jpeg` 运行时预览
- 大图等比缩小
- 小图原尺寸居中显示
- 图片预览错误原因提示

对于 `STM32F407 + FATFS + LCD` 这个级别的平台，这已经是一套可实际使用的第一版图片浏览能力。
